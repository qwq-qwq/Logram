01 · Открытие лога

Logram сразу работает с многогигабайтными файлами: на Windows используется memory-mapped I/O, на macOS — потоковое чтение в Data. Формат определяется автоматически по заголовку файла — никаких мастеров импорта не существует.

macOS Кнопка Open в тулбаре или ⌘O
Windows Меню File → Open или CtrlO

Также сработает:

  • Перетаскивание файла в окно приложения.
  • Командная строка — путь первым аргументом: Logram.exe path\app.log или open -a Logram ~/logs/app.log.
  • Ассоциация с файлами — двойной клик по .log в Finder/Explorer, если приложение назначено по умолчанию.

02 · Форматы UB-логов

Поддерживаются четыре диалекта. Строка всегда имеет структуру: [дата] [поток] [уровень] <TAB> сообщение. Номер потока кодируется одним ASCII-символом: ! = 0, " = 1, # = 2 и далее (charCode − 0x21).

mORMot 1 — основной формат UB

20210727 08260933 $ SQL SELECT id FROM uba_user WHERE name=:name

Временная метка: YYYYMMDD HHMMSSCC (сотые доли секунды в конце).

mORMot 2 HiRes — высокоточный таймер

0000000000000C72 ! info UB server started on port 8881

16-значный hex-счётчик вместо даты — удобно для микро-бенчмарков.

journald — systemd

2022-10-27T00:00:00.358664+0300 ub[657518]: $ SQL COMMIT

ISO-8601 с микросекундами и префиксом ub[pid]:.

console — stdin без даты

$ EXC EUBSecurityException at auth.js:142

Когда UB запущен в режиме разработки и пишет прямо в терминал.

Уровни

Logram распознаёт 32 уровня UB-логов: info, SQL, EXC, WRN, ERR, DBG, HTTP, +/- (вход/выход метода), MON, CAC и другие. Каждому назначен цвет badge и оттенок строки; палитру задаёт тема (п. 7).

03 · Фильтры и пресеты

В боковой панели — две группы чекбоксов и пресеты над ними:

  • Log Levels — включить/выключить строки определённого уровня. Рядом с каждым уровнем — счётчик событий.
  • Threads — показывать строки только выбранных воркеров. Удобно, когда нужно следить за одним HTTP-запросом.

Фильтрация O(n) — один проход по всем строкам: уровни → потоки → текст. На Windows используются битовые маски уровней, поэтому миллион строк перефильтровывается за ~50 мс.

Пресеты

macOS Pills Errors · SQL · HTTP · +/- над чекбоксами уровней
Windows Кнопки All · Errors · SQL · HTTP · +/- в панели фильтров

Что включает каждый пресет:

  • All (только Windows) — все уровни.
  • Errors — Error, EXC, EXC OS, OS ERR, Fail (на Windows ещё DddER). Быстро увидеть «что сломалось».
  • SQL — SQL, cust1, cust2. Обзор запросов к БД вместе с параметрами.
  • HTTP — HTTP (на Windows ещё Clnt, Srvr). Трассировка внешних запросов.
  • +/- — уровни + (вход в метод) и - (выход). Полезно перед открытием Method Timing.

Пресет обнуляет текущие чекбоксы уровней и включает только указанные. На macOS повторный клик (когда все уровни пресета уже активны) — выключает их. Текстовый фильтр и фильтр по потокам пресеты не трогают.

На macOS под списком есть кнопки Reset / Apply — Reset возвращает все чекбоксы в состояние «включено всё», Apply применяет изменения.

04 · Поиск

Находит подстроку в текущем видимом наборе строк (после фильтров). Регистронезависимый; регулярные выражения пока не поддерживаются.

macOS Поле Search в тулбаре (всегда видимое)
Windows Поле Search в тулбаре · CtrlF фокусирует поле

Навигация между совпадениями:

  • macOS — стрелки ↑/↓ в тулбаре (рядом с полем поиска) или Enter / Shift+Enter.
  • Windows — F3 / Shift+F3, либо кнопки Find Next / Find Previous в тулбаре.

При достижении края поиск закольцовывается с противоположной стороны и сопровождается коротким звуком. Если совпадений нет вообще — звук предупреждения.

Рядом с навигацией совпадений есть ещё пара стрелок с индикатором ошибки — они прыгают к следующей/предыдущей ошибке (см. Method Timing про строки Error/EXC).

05 · Method Timing

UnityBase логирует вход в метод строкой уровня + и выход — строкой - на том же потоке. Logram для каждого + находит парный - с учётом вложенности и считает длительность.

Колонка Duration

По умолчанию скрыта (построение таймингов стоит времени на больших файлах). При первом включении Logram строит тайминги и кэширует — следующие переключения мгновенные. На новом файле настройка сбрасывается.

macOS Кнопка-таймер ⏱ в тулбаре
Windows Меню View → Toggle Duration

Сводка методов

Отдельное окно с таблицей всех пар +/- и сортировкой по длительности. Двойной клик по строке — переход к соответствующему месту в логе. Можно оставить открытым рядом с главным окном.

macOS Кнопка Timing в тулбаре (справа)
Windows Меню View → Method Timing...

Прыжок к паре

Если вы стоите на строке + или -, ⌘J / CtrlJ переносит к парной строке на том же потоке. На macOS тот же переход доступен через кнопку со стрелкой ↓/↑ в тулбаре (появляется когда выбрана строка + или -).

Навигация по ошибкам

Прыжок к следующей/предыдущей строке уровня Error/EXC:

macOS Стрелки ↑/↓ возле красного ⚠︎ в тулбаре
Windows Меню Navigate → Next/Prev Error · CtrlShift↓/↑

06 · SQL и cust1

Когда вы выбираете строку уровня SQL, в панели деталей снизу запрос подсвечивается (ключевые слова, строки, числа, функции) и форматируется для читабельности. Если на том же потоке перед запросом есть строка cust1 с JSON-параметрами вида {"P1":…,"P2s8":"…"}, Logram автоматически подставляет их в плейсхолдеры :1/:2 (Oracle) либо позиционные ?.

Ключи JSON — это маркеры типа, которые генерирует UB-драйвер:

  • PN — параметр номер N (число, булево значение или null). Пример: P1 → :1.
  • PNslen — строка длиной len байт. Значение оборачивается в кавычки: P2s8: "demo user" → :2 = 'demo user'.
  • PNd — дата/время в ISO-формате. Пример: P3d: "2025-01-01" → :3 = '2025-01-01'.

То есть P2s8 означает: «параметр №2, строка из 8 символов». Сам суффикс в подстановке не участвует — Logram смотрит только на номер после буквы P.

Результат — готовый для копирования SQL, который можно выполнить в psql/DBeaver. Переключатель Show params / Hide params в панели управляет подстановкой.

JSON в деталях (например, тело HTTP-запроса) тоже форматируется pretty-print.

07 · Темы и колонки

Две встроенные темы: Tokyo Night (приглушённая, для долгих сессий) и TTY (яркая, похожая на терминал). Переключаются без перезагрузки.

macOS Иконка-палитра в тулбаре (меню с темами)
Windows Меню View → Theme → Tokyo Night / TTY

Цвета применяются к:

  • Badge уровня и оттенку строки (EXC, Error — красные; SQL — фиолетовый; Warn — жёлтый; HTTP — бирюзовый, и т.д.).
  • Колонки потока — каждый поток имеет стабильный цвет от своего номера.
  • SQL-подсветке в панели деталей (ключевые слова, строки, числа, функции).

Настройка темы запоминается между запусками. Колонка Duration — тоже, но сбрасывается в off при открытии нового файла.

08 · Вкладки и окна

macOS Нативные вкладки macOS · ⌘N — новая вкладка · кнопка + в титульной полосе
Windows Каждый Logram.exe — отдельное окно

macOS: вкладки работают через WindowGroup. ⌘O открывает файл в активной вкладке (фокус маршрутизируется через @FocusedValue). Вкладки перетаскиваются в отдельные окна и обратно.

Windows: открывайте параллельно сколько нужно — благодаря memory-mapped I/O стоимость дополнительной копии лога мала.
