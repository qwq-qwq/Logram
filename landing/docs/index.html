01 · Відкриття логу

Logram одразу працює з багатогігабайтними файлами: на Windows використовує memory-mapped I/O, на macOS — поточне читання в Data. Формат визначається автоматично за заголовком файлу — жодних майстрів імпорту не існує.

macOS Кнопка Open у тулбарі або ⌘O
Windows Меню File → Open або CtrlO

Також спрацює:

  • Перетягування файлу у вікно застосунку.
  • Командний рядок — шлях першим аргументом: Logram.exe path\app.log або open -a Logram ~/logs/app.log.
  • Асоціація з файлами — подвійний клік на .log у Finder/Explorer, якщо застосунок призначено типовим.

02 · Формати UB-логів

Підтримуються чотири діалекти. Рядок завжди має структуру: [дата] [потік] [рівень] <TAB> повідомлення. Номер потоку кодується одним ASCII-символом: ! = 0, " = 1, # = 2 і далі (charCode − 0x21).

mORMot 1 — основний формат UB

20210727 08260933 $ SQL SELECT id FROM uba_user WHERE name=:name

Часова мітка: YYYYMMDD HHMMSSCC (сотні секунди в кінці).

mORMot 2 HiRes — високоточний таймер

0000000000000C72 ! info UB server started on port 8881

16-значний hex-лічильник замість дати — зручно для мікро-бенчмарків.

journald — systemd

2022-10-27T00:00:00.358664+0300 ub[657518]: $ SQL COMMIT

ISO-8601 з мікросекундами і префіксом ub[pid]:.

console — stdin без дати

$ EXC EUBSecurityException at auth.js:142

Коли UB запущений у режимі розробки і пише прямо в термінал.

Рівні

Logram розпізнає 32 рівня UB-логів: info, SQL, EXC, WRN, ERR, DBG, HTTP, +/- (вхід/вихід методу), MON, CAC та інші. Кожному призначено колір badge та відтінок рядка; палітру задає тема (п. 7).

03 · Фільтри й пресети

У боковій панелі — дві групи чекбоксів і пресети над ними:

  • Log Levels — увімкнути/вимкнути рядки певного рівня. Поруч із кожним рівнем — лічильник подій.
  • Threads — показувати рядки тільки обраних воркерів. Зручно, коли треба стежити за одним HTTP-запитом.

Фільтрація O(n) — один прохід по всіх рядках: рівні → потоки → текст. На Windows використовуються бітові маски рівнів, тому мільйон рядків перефільтровується за ~50 мс.

Пресети

macOS Pills Errors · SQL · HTTP · +/- над чекбоксами рівнів
Windows Кнопки All · Errors · SQL · HTTP · +/- у панелі фільтрів

Що вмикає кожен пресет:

  • All (лише Windows) — усі рівні.
  • Errors — Error, EXC, EXC OS, OS ERR, Fail (на Windows ще DddER). Швидко побачити «що зламалось».
  • SQL — SQL, cust1, cust2. Огляд запитів до БД разом із параметрами.
  • HTTP — HTTP (на Windows ще Clnt, Srvr). Трасування зовнішніх запитів.
  • +/- — рівні + (вхід у метод) та - (вихід). Корисно перед відкриттям Method Timing.

Пресет обнуляє поточні чекбокси рівнів і вмикає тільки зазначені. На macOS повторний клік (коли всі рівні пресета вже активні) — їх вимикає. Текстовий фільтр і фільтр за потоками пресети не чіпають.

На macOS під списком є кнопки Reset / Apply — Reset повертає всі чекбокси до стану «увімкнено все», Apply застосовує зміни.

04 · Пошук

Знаходить підрядок у поточному видимому наборі рядків (після фільтрів). Регістронезалежний; регулярні вирази поки не підтримуються.

macOS Поле Search у тулбарі (завжди видиме)
Windows Поле Search у тулбарі · CtrlF фокусує поле

Навігація між збігами:

  • macOS — стрілки ↑/↓ у тулбарі (поруч із полем пошуку) або Enter / Shift+Enter.
  • Windows — F3 / Shift+F3, або кнопки Find Next / Find Previous у тулбарі.

При досягненні краю пошук обертається з протилежного боку й супроводжується коротким звуком. Якщо збігів немає взагалі — звук попередження.

Поруч із навігацією збігів є ще пара стрілок з індикатором помилки — вони стрибають до наступної/попередньої помилки (див. Method Timing про рядки Error/EXC).

05 · Method Timing

UnityBase логує вхід у метод рядком рівня + і вихід — рядком - на тому ж потоці. Logram для кожного + знаходить парний - з урахуванням вкладеності і рахує тривалість.

Колонка Duration

За замовчуванням прихована (побудова таймінгів коштує часу на великих файлах). При першому ввімкненні Logram будує тайминги й кешує — наступні перемикання миттєві. На новому файлі налаштування скидається.

macOS Кнопка-таймер ⏱ у тулбарі
Windows Меню View → Toggle Duration

Зведення методів

Окреме вікно з таблицею всіх пар +/- і сортуванням за тривалістю. Подвійний клік по рядку — перехід до відповідного місця в логу. Можна залишити відкритим поруч із головним вікном.

macOS Кнопка Timing у тулбарі (праворуч)
Windows Меню View → Method Timing...

Стрибок до пари

Якщо ви стоїте на рядку + або -, ⌘J / CtrlJ переносить до парного рядка на тому ж потоці. На macOS той самий перехід доступний через кнопку зі стрілкою ↓/↑ у тулбарі (з'являється коли обрано рядок + або -).

Навігація по помилках

Стрибок до наступного/попереднього рядка рівня Error/EXC:

macOS Стрілки ↑/↓ біля червоного ⚠︎ у тулбарі
Windows Меню Navigate → Next/Prev Error · CtrlShift↓/↑

06 · SQL і cust1

Коли ви обираєте рядок рівня SQL, у панелі деталей знизу запит підсвічується (ключові слова, рядки, числа, функції) і форматується для читабельності. Якщо на тому ж потоці перед запитом є рядок cust1 із JSON-параметрами виду {"P1":…,"P2s8":"…"}, Logram автоматично підставляє їх у плейсхолдери :1/:2 (Oracle) або позиційні ?.

Ключі JSON — це маркери типу, які генерує UB-драйвер:

  • PN — параметр номер N (число, булеве значення або null). Приклад: P1 → :1.
  • PNslen — рядок довжиною len байтів. Значення обгортається в лапки: P2s8: "demo user" → :2 = 'demo user'.
  • PNd — дата/час у ISO-форматі. Приклад: P3d: "2025-01-01" → :3 = '2025-01-01'.

Тобто P2s8 означає: «параметр №2, рядок з 8 символів». Сам суфікс у підстановці не бере участі — Logram дивиться лише на номер після літери P.

Результат — готовий для копіювання SQL, який можна виконати в psql/DBeaver. Перемикач Show params / Hide params у панелі керує підстановкою.

JSON у деталях (наприклад, тіло HTTP-запиту) теж форматується pretty-print.

07 · Теми та колонки

Дві вбудовані теми: Tokyo Night (приглушена, для довгих сесій) і TTY (яскрава, схожа на термінал). Перемикаються без перезавантаження.

macOS Іконка-палітра у тулбарі (меню з темами)
Windows Меню View → Theme → Tokyo Night / TTY

Кольори застосовуються до:

  • Badge рівня та відтінку рядка (EXC, Error — червоні; SQL — фіолетовий; Warn — жовтий; HTTP — бірюзовий, і т.д.).
  • Колонки потоку — кожен потік має стабільний колір від свого номера.
  • SQL-підсвічування в панелі деталей (ключові слова, рядки, числа, функції).

Налаштування теми запам'ятовується між запусками. Колонка Duration — також, але скидається в off при відкритті нового файлу.

08 · Вкладки й вікна

macOS Нативні вкладки macOS · ⌘N — нова вкладка · кнопка + у титульній смузі
Windows Кожен Logram.exe — окреме вікно

macOS: вкладки працюють через WindowGroup. ⌘O відкриває файл у активній вкладці (фокус маршрутизується через @FocusedValue). Вкладки перетягуються в окремі вікна й назад.

Windows: відкривайте паралельно скільки потрібно — завдяки memory-mapped I/O вартість додаткової копії логу мала.
