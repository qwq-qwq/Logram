Documentation.

A short reference: how to open files, manage filters, find slow methods, read SQL with substituted cust1 parameters, and which keys do what.

01 · Opening a log

Logram works with multi-gigabyte files out of the box: Windows uses memory-mapped I/O, macOS — streaming read into Data. The format is auto-detected from the file header — no import wizards involved.

macOS Open button in the toolbar or ⌘O
Windows File → Open menu or CtrlO

These also work:

  • Drag and drop a file onto the app window.
  • Command line — path as the first argument: Logram.exe path\app.log or open -a Logram ~/logs/app.log.
  • File association — in Finder/Explorer right-click a .log file and pick Logram from Open With…; or set Logram as the default handler for double-click.

02 · UB log formats

Four dialects are supported. A line always has the structure: [date] [thread] [level] <TAB> message. The thread number is encoded as a single ASCII character: ! = 0, " = 1, # = 2 and so on (charCode − 0x21).

mORMot 1 — the main UB format

20210727 08260933 $ SQL SELECT id FROM uba_user WHERE name=:name

Timestamp: YYYYMMDD HHMMSSCC (hundredths of a second at the end).

mORMot 2 HiRes — high-resolution timer

0000000000000C72 ! info UB server started on port 8881

A 16-digit hex counter instead of a date — handy for micro-benchmarks.

journald — systemd

2022-10-27T00:00:00.358664+0300 ub[657518]: $ SQL COMMIT

ISO-8601 with microseconds and the ub[pid]: prefix.

console — stdin without a date

$ EXC EUBSecurityException at auth.js:142

When UB runs in development mode and writes straight to the terminal.

Levels

Logram recognises 32 UB log levels: info, SQL, EXC, WRN, ERR, DBG, HTTP, +/- (method enter/exit), MON, CAC and others. Each one has a badge colour and a row tint; the palette is set by the theme (section 7).

03 · Filters & presets

The sidebar has two checkbox groups and presets above them:

  • Log Levels — show/hide rows of a given level. Each level shows a hit counter next to it.
  • Threads — show rows only from selected workers. Useful when tracing a single HTTP request.

Filtering is O(n) — a single pass over all rows: levels → threads → text. Windows uses level bitmasks, so a million rows re-filter in ~50 ms.

Presets

macOS Pills Errors · SQL · HTTP · +/- above the level checkboxes
Windows Buttons All · Errors · SQL · HTTP · +/- in the filter panel

What each preset enables:

  • All (Windows only) — every level.
  • Errors — Error, EXC, EXC OS, OS ERR, Fail (plus DddER on Windows). Quickly spot "what broke".
  • SQL — SQL, cust1, cust2. DB query overview together with parameters.
  • HTTP — HTTP (plus Clnt, Srvr on Windows). Tracing outbound requests.
  • +/- — levels + (method enter) and - (exit). Useful before opening Method Timing.

A preset resets the current level checkboxes and enables only the listed ones. On macOS, clicking the same preset again (when all its levels are already active) disables them. The text filter and thread filter are not touched by presets.

On macOS, Reset / Apply buttons sit below the list — Reset sets every checkbox back to "all on", Apply commits the change.

04 · Search

Finds a substring in the currently visible set of rows (after filters). Case-insensitive; regular expressions are not supported yet.

macOS Search field in the toolbar (always visible)
Windows Search field in the toolbar · CtrlF focuses the field

Navigating between matches:

  • macOS — arrows ↑/↓ in the toolbar (next to the search field) or Enter / Shift+Enter.
  • Windows — F3 / Shift+F3, or the Find Next / Find Previous buttons in the toolbar.

When the end is reached, search wraps around from the opposite side and plays a short beep. If there are no matches at all — a warning beep.

Next to the match navigation there is another pair of arrows with an error indicator — they jump to the next/previous error (see Method Timing regarding Error/EXC rows).

05 · Method Timing

UnityBase logs a method entry with a +-level line and the exit with a - line on the same thread. For every +, Logram finds its matching -, respecting nesting, and computes the duration.

Duration column

Hidden by default (building timings costs time on large files). The first time you enable it, Logram computes and caches the timings — subsequent toggles are instant. The setting resets on a new file.

macOS Timer button ⏱ in the toolbar
Windows View → Toggle Duration menu

Method summary

A separate window with a table of all +/- pairs sorted by duration. Double-click a row to jump to that place in the log. Can be left open next to the main window.

macOS Timing button in the toolbar (right side)
Windows View → Method Timing... menu

Jump to pair

If you're on a + or - row, ⌘J / CtrlJ jumps to the paired line on the same thread. On macOS the same jump is available via an arrow button ↓/↑ in the toolbar (appears when a + or - row is selected).

Error navigation

Jump to the next/previous Error/EXC line:

macOS Arrows ↑/↓ next to the red ⚠︎ in the toolbar
Windows Navigate → Next/Prev Error menu · CtrlShift↓/↑

06 · SQL and cust1

When you select a SQL-level row, the query is highlighted in the detail panel below (keywords, strings, numbers, functions) and formatted for readability. If there's a preceding cust1 line on the same thread with a JSON object like {"P1":…,"P2s8":"…"}, Logram automatically substitutes the values into :1/:2 (Oracle) or positional ? placeholders.

JSON keys are type markers produced by the UB driver:

  • PN — parameter number N (number, boolean, or null). Example: P1 → :1.
  • PNslen — string of len bytes. The value is wrapped in quotes: P2s8: "demo user" → :2 = 'demo user'.
  • PNd — date/time in ISO format. Example: P3d: "2025-01-01" → :3 = '2025-01-01'.

So P2s8 means "parameter #2, an 8-character string". The suffix itself isn't used for substitution — Logram only looks at the number after the P.

The result is a copy-ready SQL you can run in psql/DBeaver. The Show params / Hide params toggle in the panel controls substitution.

JSON in details (e.g. HTTP body) is pretty-printed too.

07 · Themes & columns

Two built-in themes: Tokyo Night (muted, for long sessions) and TTY (bright, terminal-like). Switched without a restart.

macOS Palette icon in the toolbar (theme menu)
Windows View → Theme → Tokyo Night / TTY menu

Colours are applied to:

  • Level badge and row tint (EXC, Error — red; SQL — purple; Warn — yellow; HTTP — teal, etc.).
  • Thread column — each thread gets a stable colour derived from its number.
  • SQL highlighting in the detail panel (keywords, strings, numbers, functions).

The theme choice is remembered between runs. The Duration column too — but it resets to off whenever a new file is opened.

08 · Tabs & windows

macOS Native macOS tabs · ⌘N — new tab · + button in the title bar
Windows Each Logram.exe is a separate window

macOS: tabs work via WindowGroup. ⌘O opens a file in the active tab (focus is routed through @FocusedValue). Tabs can be dragged into standalone windows and back.

Windows: open as many in parallel as you need — thanks to memory-mapped I/O the cost of an extra log copy is low.
